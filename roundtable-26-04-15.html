Most people think Python is "interpreted." It is - but it's compiled first. Just not to machine code. It's compiled to bytecode, which is then interpreted by the VM. The key insight: by the time your code runs, it doesn't look anything like what you typed. dis.dis() is genuinely fun to play with - encourage people to try it. The five-stage pipeline mirrors the CPython internals reference: tokenize, parse to AST, optimize, compile to bytecode, cache to .pyc.

These two stages aren't sequential - they run in parallel beneath everything else. The object layer is the single most important concept: everything is a PyObject. When someone says "Python is slow because everything is an object" - this is literally what they mean. Every integer carries a refcount and a type pointer. The GIL is the thing everyone has an opinion about and half the room has never actually been bitten by. The free-threaded build is the biggest change to CPython in decades.

Honourable Mentions

  • It’s all C under the hood - CPython is ~500k lines of C compiled to a native binary. Your Python never becomes machine code; it stays as bytecode that this C program interprets.
  • CPython is not the only Python - PyPy (JIT), Cython (AOT to C), Jython (JVM), IronPython (.NET). Everything in this talk is specific to CPython - the reference implementation that 95%+ of people use.
  • What’s new in 3.13+ / 3.14 - Free-threaded mode (PEP 703) removes the GIL with per-object locks. Copy-and-patch JIT stitches pre-compiled machine code templates for hot bytecodes. Both opt-in and experimental.
  • You can inspect all of this yourself - dis.dis(fn), ast.dump(ast.parse(src)), sys.getrefcount(obj), id(obj), type(obj).__mro__, gc.get_referrers(obj). Python gives you introspection into almost everything in the pipeline.
  • A note on terminology - The Python VM is the abstract machine (eval loop + value stack + frame stack). CPython is the concrete C implementation of that VM plus everything around it (compiler, object system, GC, GIL). The VM is the spec; CPython is the reality.

So... Did We Need to Know Any of That?

Joel Spolsky’s Law: all non-trivial abstractions leak. The question is when.
